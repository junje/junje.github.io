环形链表

题目描述

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

进阶：

你能用 O(1)（即，常量）内存解决此问题吗？

示例 1：

1
2
3

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。

题目解析

此题2种解法，一种是有hashSet，存储每一个节点，如果存储失败，代表集合内存在相同节点，就代表有环了。空间复杂度On。

第二种是用快慢指针，空间复杂度O(1)。思路：可以想象成龟兔赛跑，当兔子追上乌龟后，就代表有环。乌龟=满指针，兔子=快指针，

每次让乌龟前进一步，兔子前进2步，当乌龟和兔子相等代表有环。 注意点：慢指针起始点是head，快指针起始点是head.next，因为用的是while循环，如果起始点一样，直接就返回有环了，所以要错开。

// HashSet解法
public boolean hasCycle(ListNode head) {
    Set<ListNode> set = new HashSet();
    while (head != null) {
        if (!set.add(head)) {
            return true;
        }
        head = head.next;
    }
    return false;

}

// 快慢指针解法
public boolean hasCycle(ListNode head) {
    if (head == null || head.next == null) {
        return false;
    }

    ListNode slow = head;
    ListNode fast = head.next;

    while (slow != fast) {
        if (fast == null || fast.next == null) {
            return false;
        }
        slow = slow.next;
        fast = fast.next.next;
    }

    return true;
}
